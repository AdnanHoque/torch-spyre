## Interpretation

For the tested shape `(B=1, H=2, L=128, D=64, block=64)`, the compiler emits
two flash tiles.  The likely realizer targets are:

- each `batchmatmul -> maxnonstick/sub/exp` score-tile chain;
- each `exp_scores -> batchmatmul(PV)` tile chain;
- the accumulation chain around `add`, `mul`, `sumnonstick`, and `realdiv`.

The immediate production-safe next step is not full overlap.  It is a serial
mixed SuperDSC fold for one tile chain, using the Stage 009 wrapper and the
current `STCDPOpLx` bridge, with overlap still disabled.  Once that produces
value-correct device output and clean `senprog.txt`, the same realizer can try
`SPYRE_FLASH_ATTENTION_MIXED_PIPELINE_OVERLAP=1`.

## Conclusion

The mixed-pipeline flag now demonstrably reaches the flash-prefill lowering on
device.  The next implementation should consume the concrete SDSC graph above
and fold a real tile chain into the production-shaped mixed artifact introduced
in Stage 009.
