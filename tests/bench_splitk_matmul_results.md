# SplitK matmul perf + accuracy bench — Phase 1

PyTorch:        2.10.0+cpu
torch_spyre:    (editable)
SENCORES:       32 (default)
warmup iters:   5
measure iters:  20

**Timing**: per-iter `time.perf_counter()` around `mm_fn` + `torch_spyre.streams.synchronize()`. `.to('cpu')` happens AFTER the timed region for correctness check only — D2H is not in the kernel wall-time.

**TFLOPs/s** = 2·M·N·K / median_time. End-to-end including any cross-core reduction (the dxp_standalone backend handles K-split partial-sum reduction; we observe its cost in the wall-time).

**Drift** is vs fp32 CPU reference (`a.to(fp32) @ b.to(fp32)`). Relative error masks |ref| ≤ 1e-3.
