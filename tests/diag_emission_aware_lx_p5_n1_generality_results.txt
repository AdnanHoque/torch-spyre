## Reading guide

Per-shape pattern to look for:
  pure-M wall ≈ HMI/compute baseline
  (16, 1, 2)+kf and (8, 1, 4)+kf walls ≤ ~2× pure-M  →  fast path active
  (1, 1, 32)+kf wall ≈ allreduce regime
  (1, 8, 4)+kf wall >> 2× pure-M (catastrophic)  →  confirms n>1 penalty

Generality verdict:
  All 4 shapes show pattern  →  n=1 fast path is general
  Pattern only on o_proj-like shapes  →  shape-specific
  Mixed  →  characterise the trigger condition
