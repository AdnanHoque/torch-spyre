## Reading guide

  - If BW spread per shape is < 5 GB/s: access pattern doesn't matter.
    The 40 GB/s ceiling lives in the kernel template / runtime.
    No torch_spyre lever to close the gap.
  - If BW spread > 10 GB/s: access pattern matters. The planner could
    bias toward higher-BW splits. torch_spyre lever exists.
  - If pure-K consistently wins big: K-split shrinks per-core HMI demand
    in a way the broadcast model under-counts.
