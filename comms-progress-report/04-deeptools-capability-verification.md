## Notes / cautions

- **The team's "it already exists" claim is half-true and lane-sensitive.** The *primitives* (`STCDPOpLx` scatter, multicast `prodConsList`, `ReStickifyOpLx` op, `AllGather`/`AllReduce`, `GatherOpHBM`, PSUM reduce) all exist as live code. But only two lanes are reachable *from the dl-dsc coordinate contract without an HBM round-trip*: **1:1 scatter** and **1:many multicast/broadcast**, both via the single `insertRelayoutSdsc → STCDPOpLx` path. Everything the stcdp-agent notes attribute to the remaining Granite gap is genuinely unreachable in-lane.

- **The stcdp-agent's central backend claim checks out precisely.** `InputFetchNeighbor` machinery is real and would produce the loop-scoped operand movement they want, but (a) its only caller is a standalone tool — it is **not wired into the compile pipeline**, and (b) it is hard-coded to `DsTypes::INPUT`, while the Granite value-side operand is `DsTypes::KERNEL`. Both halves of "the machinery exists but is not generalized to fetch a specific matmul operand ordinal / ds type" are verified. Verdict: **present-but-dead** for this use, not absent — the fix is DXP wiring + a `DsTypes` generalization, not a new ring primitive.

- **`ReStickifyOpLx` is subtler than "live."** The op runs, but the auto-insertion pass (`insertRelayoutSdsc`) never *chooses* it — it only emits `STCDPOpLx`. The dldsc-agent's prototype emits `ReStickifyOpLx` from Torch, which lands on a real op, but no backend path derives the differing pre/post stick-layout forms `SdscRelayoutInsertion` assumes same-layout (input/output LDS treated as the consumer form). So the layout-restickify lane is **op-present, contract-absent** — matching the dldsc-agent's own assessment that a pre/post layout field is missing.

- **Reduce/all-reduce should be stated, not designed** (consistent with out-of-scope framing): the on-chip reduce is PSUM/array-local compute; the cross-device `coll`/multi-AIU `AllReduce` is a different subsystem and does not offer a single-AIU LX relayout hook.

- **Status honesty:** scatter = implemented+measured (live path). Multicast = live primitive, but not yet driven from the dl-dsc classifier for the Granite operand (prototype/classified-only). InputFetchNeighbor generalization, `ReStickifyOpLx` auto-insertion, and any move-then-reduce = future/backend-gap, not done.
